name="what" id="what"></a>             <p><strong>What is the NDFD REST Service?</strong></p>             <p><a href="http://www.nws.noaa.gov/ndfd/">National Digital Forecast Database</a> (NDFD) REST                  service provides the public, government agencies, and commercial                 enterprises with data from the National Weather Service&rsquo;s (NWS) digital forecast                 database. &nbsp; This service, which is defined in a                 <a href="http://products.weather.gov/PDD/Extensible_Markup_Language.pdf">Service Description Document</a>,                 provides NWS customers and partners the ability to retrieve                  <a href="http://www.nws.noaa.gov/ndfd/technical.htm">NDFD data</a> over the internet that is                encoded in XML.</p>              <p>The <a href="http://forecasts.weather.gov/cgi-bin/nwsexit.pl?url=http://www.w3.org/XML/">XML</a> dialect used is called                 Digital Weather Markup Language (DWML) and its schema can be found at the following link:</p>             <p><a href="http://graphical.weather.gov/xml/DWMLgen/schema/DWML.xsd">http://graphical.weather.gov/xml/DWMLgen/schema/DWML.xsd</a></p>             <p>The schema defines the XML language&rsquo;s vocabulary (available tags and attributes) and                 syntax (expected order of tags). &nbsp; Knowing how DWML is constructed, users can write                 programs to process the XML document to get at the data it contains.</p>              <p>To see a sample Digital Weather Markup Language document that conforms to the DWML schema,                 click on any of the following links:</p>             <p><a href="http://graphical.weather.gov/xml/DWMLgen/schema/latest_DWML.txt">http://graphical.weather.gov/xml/DWMLgen/schema/latest_DWML.txt</a><br />               <a href="http://graphical.weather.gov/xml/DWMLgen/schema/latest_DWML_glance.txt">http://graphical.weather.gov/xml/DWMLgen/schema/latest_DWML_glance.txt</a><br />               <a href="http://graphical.weather.gov/xml/DWMLgen/schema/latest_DWMLByDay24hr.txt">http://graphical.weather.gov/xml/DWMLgen/schema/latest_DWMLByDay24hr.txt</a><br />               <a href="http://graphical.weather.gov/xml/DWMLgen/schema/latest_DWMLByDay12hr.txt">http://graphical.weather.gov/xml/DWMLgen/schema/latest_DWMLByDay12hr.txt</a></p>              <a name="why" id="why"></a>             <p><strong>Why provide an NDFD REST service?</strong></p>             <p>The National Weather Service is striving to serve society&rsquo;s needs for weather                 information by evolving its services from a text-based paradigm to one based on making                 NWS information available quickly, efficiently, and in convenient and understandable                 forms. &nbsp; The NDFD is one example of this transformation. &nbsp; NDFD REST takes yet                 another step towards a digital services era by making NDFD data available for computer to                 computer transfer and processing. &nbsp; NWS customers and partners can then enhance the                 value of NDFD data through the creation of value added products.</p>               <p>You can provide comments on this web service from the following page.<br /><br />                <a href="http://www.nws.noaa.gov/feedback.php">http://www.nws.noaa.gov/feedback.php</a></p>              <a name="frequency" id="frequency"></a>             <p><strong>How often is NDFD data updated?</strong></p>             <p>The NDFD data available via the REST service is updated no more than hourly. As a result, we                 request developers using this REST service only make a request for a specific point no more                 than once an hour. The database is currently updated by 45 minutes after the hour.</p>              <a name="generate_it" id="generate_it"></a>             <p>You can interact with the REST service by visiting the following URLs:</p>             <p><a href="http://graphical.weather.gov/xml/sample_products/browser_interface/ndfdXML.htm">http://graphical.weather.gov/xml/sample_products/browser_interface/ndfdXML.htm</a>                <a href="http://graphical.weather.gov/xml/sample_products/browser_interface/ndfdBrowserByDay.htm">http://graphical.weather.gov/xml/sample_products/browser_interface/ndfdBrowserByDay.htm</a></p>              <p>By clicking the &ldquo;submit&rdquo; button on the above web pages, a script is executed to                 request and retrieve DWML-encoded NDFD data.</p>              <a name="XML_contents" id="XML_contents"></a>             <p><strong>What data does the NDFD contain?</strong></p>             <p>NDFD XML contains forecasts for any combination of the following meteorological parameters:</p>             <p><a href="http://www.nws.noaa.gov/ndfd/technical.htm#elements">                         http://www.nws.noaa.gov/ndfd/technical.htm#elements</a></p>              <p>For additional information on NDFD parameter definitions and available time projection, please                 see the <a href="http://www.nws.noaa.gov/ndfd/technical.htm">NDFD technical page</a>.</p>              <p>These NDFD parameters can be retrieved in their unsummarized form using the ndfdXMLclient.php interface, or                 summarized over either 24- or 12-hourly periods using the ndfdBrowserClientByDay.php interface.                 See the section below titled &quot;How Do You Use the Web Service?&quot; for more details on                 these two methods of accessing the service.</p>              <p>In addition to its meteorological content, the NDFD REST service response contains meta data.                 &nbsp; A description of the meta data and data XML tags and attributes can be                 found in the                 <a href="http://graphical.weather.gov/xml/mdl/XML/Design/MDL_XML_Design.pdf">Digital Weather Markup Language Specification</a>                <a href="http://graphical.weather.gov/xml/mdl/XML/Design/MDL_XML_Design.pdf">[PDF]</a>                <a href="http://graphical.weather.gov/xml/mdl/XML/Design/MDL_XML_Design.htm">[HTML]</a>                <a href="http://graphical.weather.gov/xml/mdl/XML/Design/MDL_XML_Design.doc">[DOC]</a></p>              <a name="use_it" id="use_it"></a>             <p><strong>How Do You Use the Web Service? </strong></p>             <p>The first step to using the web service is to create a RESTful client. &nbsp; Degrib is a C program that allows you to visualize NDFD GRIB2 data and convert these binary files into other  formats.  If you are a web service user who wants DWML-encoded data for a large number of NDFD points, degrib will  allow you to move the DWML-encoding process to your computer.  By using degrib to process the NDFD data on a local  computer, you may be able to significantly improve the speed with which your program is able to ingest NDFD data. The NWS makes a degrib executable available for Microsoft Windows PCs and the degrib source code available for compilation on other computer operating systems.  To download degrib software, visit the URL <a href="http://www.nws.noaa.gov/mdl/degrib/register.php">http://www.nws.noaa.gov/mdl/degrib/register.php</a>.
