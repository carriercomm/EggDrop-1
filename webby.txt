You can still encrypt the device after doing this, but it won't be automatically done.<br /> <br /> After observing how this force encryption stuff works, I got it mostly figured out. (It's entirely a SW layer, as is already widely known). Basically when all the devices from fstab are mounted in android with the forceencrypt option, fs_mgr sets a flag for encryption (something like IF This_Device_Isnt_Encrypted; then This_Device_Needs_Encryption). on devices (looks like android only allows you to encrypt 1 device, which is probably to prevent such cases as over-resource usage ,maybe some other conflict that it doesn't support over 1 device, idk) that have forceencrypt set on them, if it can't unmount the device before doing these encryption checks - in other words if it's b usy (like a file is open) - it just skips encryption all together. So if the device had a file preventing it from being unmounted, it just says &quot;oh well, skip encryption.&quot; I found this kinda odd behavior anyway :P<br /> <br /> You can still encrypt the device, it just isn't forced. Some people are complaining about the slowness of the encryption SW-layer (why force SW encryption? At least put some HW for it in the device). This makes it the way it probably should be - optional.<br /> <br /> Stock LRX21O - No force encrypt:<br /> <a href="https://www.androidfilehost.com/?fid=95784891001613336" target="_blank" rel="nofollow" >https://www.androidfilehost.com/?fid=95784891001613336</a><br /> <br /> Stock boot.img - if you want to revert back to stock for some reason.<br /> <a href="https://www.androidfilehost.com/?fid=95784891001613337" target="_blank" rel="nofollow" >https://www.androidfilehost.com/?fid=95784891001613337</a><br /> <br /> <b>It doesn't matter if you already have root, twrp, or whatever. Do all that before, during, after whatever floats your boat it all works the same</b><br /> How-to:<br /> 1.) Reboot to boot loader<br /> 2.) Unlock device if not already (fastboot oem unlock) - will wipe all data<br /> 3.) I think unlocking the device will automatically run encryption jobs, so don't boot android write your most important stuff down and then continue without backing it up.<br /> 4.) Download  the boot.img noencrypt above<br /> 5.) Flash it in the bootloader (fastboot flash boot boot_noforceencrypt.img)<br /> 6.) To disable encryption, a factory reset is needed (Done by fastboot format userdata, once booted in Settings -&gt; Backup &amp; Reset, or in recovery)<br /> 7.) If it doesn't work, you can go back into the bootloader and flash the stock image.<br /> 8.) Run CF-Auto-Root for root, if desired. Flash TWRP, do wutever you want.<br /> 9.) Updates will overwrite this and turn encryption back on, you should probably update manually (or remove the boot.img from the update) if you don't want encryption to get turned back on (turning it back on may be a PITA in the sense that you'll have to factory reset to get it off again)<br /> <br /> I don't own a N6 at this point, just trying to help out as I was curious about the results as
