<header class="entry-header">     <h1 class="entry-title">How to develop a Firefox extension</h1>            <address class="vcard"><cite class="author fn"><a class="url" href="http://blog.mozilla.org/addons/author/jvillalobosmozilla-com/" title="See all 192 posts by Jorge Villalobos">Jorge Villalobos <img alt='' src='http://0.gravatar.com/avatar/6d1966118f16e4b99a6e3ad07883be33?s=24&amp;d=http%3A%2F%2F0.gravatar.com%2Favatar%2Fad516503a11cd5ca435acc9bb6523536%3Fs%3D24&amp;r=G' class='avatar avatar-24 photo' height='24' width='24' /></a></cite></address>                <p class="entry-comments"><a href="http://blog.mozilla.org/addons/2014/06/05/how-to-develop-firefox-extension/#comments" title="26 responses">26</a></p>                <p class="entry-posted">         <time class="published" title="2014-06-05T09:00:21-08:00" datetime="2014-06-05T09:00:21-08:00">           <a class="posted-month" href="http://blog.mozilla.org/addons/2014/06/" title="See all posts from June, 2014">Jun</a>           <span class="posted-date">5</span>           <a class="posted-year" href="http://blog.mozilla.org/addons/2014/" title="See all posts from 2014">2014</a>         </time>       </p>                </header><!-- .entry-header -->    <div class="entry-content">           <p>It&#8217;s been a while since we published a guide on extension development in this blog, and we recently discovered that many of you are hitting a very old blog post about it. There&#8217;s plenty of documentation out there, but it can be hard to find, so here&#8217;s an overview of what you need to know about extension development.</p> <p>As with everything Mozilla, the first place you should look for developer documentation is the Mozilla Developer Network. There&#8217;s an <a href="https://developer.mozilla.org/Add-ons" target="_blank"><strong>add-ons section</strong></a> there where you will find everything you need to know about how to develop Firefox extensions, as well as other add-on types.</p> <p>There are a couple different ways to go about creating an extension for Firefox, so I covered each in the sections below.</p> <h3>Add-ons SDK extensions</h3> <p>The <strong><a href="https://developer.mozilla.org/en-US/Add-ons/SDK" target="_blank">Add-ons SDK</a></strong> is a set of simple APIs you can use to quickly build good Firefox extensions. It&#8217;s what we recommend for new developers. It should be easy to work with, especially if you&#8217;re familiar with Google Chrome extensions or userscripts. It abstracts away most of the XUL / XPCOM infractrusture in Firefox, giving you a more familiar HTML and JS environment to work with.</p> <p>The current approach to building an SDK extension is to download and set up the SDK, code locally,  then package your extension (an XPI file) using the cfx tool included with the SDK. The SDK team is working on making this process much easier by integrating it into Firefox developer tools. Soon you&#8217;ll be able to build your extensions very quickly, right from Firefox.</p> <p>You can learn more about the SDK and get help here:</p> <ul> <li><a href="https://developer.mozilla.org/en-US/Add-ons/SDK" target="_blank">Documentation on MDN.</a></li> <li><a href="https://blog.mozilla.org/addons/category/sdk/" target="_blank">Articles in this blog.</a></li> <li><a href="http://stackoverflow.com/questions/tagged/firefox-addon-sdk" target="_blank">Stack Overflow.</a></li> <li><a href="https://forums.mozilla.org/addons/viewforum.php?f=27" target="_blank">Add-ons forum.</a></li> <li>#jetpack channel on <a href="https://wiki.mozilla.org/IRC" target="_blank">Mozilla IRC.</a></li> </ul> <h3>Bootstrapped extensions</h3> <p><a href="https://developer.mozilla.org/en-US/Add-ons/Bootstrapped_extensions" target="_blank"><strong>Bootstrapped extensions</strong></a> don&#8217;t require a restart to be installed, like SDK extensions, but they don&#8217;t have the easy access to SDK APIs or sandboxing. You&#8217;re basically doing everything manually, like <a href="http://www.oxymoronical.com/blog/2011/01/Playing-with-windows-in-restartless-bootstrapped-extensions" target="_blank">tracking windows to add or remove your UI</a>. However, there are various great tools available to you via <a href="https://developer.mozilla.org/en-US/docs/Mozilla/JavaScript_code_modules" target="_blank">existing JavaScript Modules</a>, like <a href="https://developer.mozilla.org/en-US/docs/Mozilla/JavaScript_code_modules/CustomizableUI.jsm" target="_blank">CustomizableUI.jsm</a> for toolbar UI and <a href="https://developer.mozilla.org/en-US/docs/Mozilla/JavaScript_code_modules/Services.jsm" target="_blank">Services.jsm</a> for frequently used Firefox components.</p> <p>Compared to the old way of making extensions (see <em>Overlay extensions</em> below), the only notable loss are overlays. Bootstrapped extensions have the bootstrap.js file instead, which is what serves as the starting point for your code. Otherwise, everything should be familiar to you: <a href="https://developer.mozilla.org/en-US/Add-ons/Install_Manifests" target="_blank">install.rdf</a>, <a href="https://developer.mozilla.org/en-US/docs/Chrome_Registration" target="_blank">chrome.manifest</a> and code files, all packed in a ZIP file with a <em>.xpi</em> extension.</p> <p>Other than <strong><a href="https://developer.mozilla.org/en-US/Add-ons/Bootstrapped_extensions" target="_blank">Bootstrapped extensions</a></strong> on MDN and the pages they link to, all the available developer resources are the same for overlay extensions (see below).</p> <h3>Firefox for Android extensions</h3> <p><strong><a href="https://developer.mozilla.org/en-US/Add-ons/Firefox_for_Android" target="_blank">Extensions in Firefox for Android</a></strong> are slightly different. Since the UI is native instead of XUL, the way extensions can modify it is different and a bit more limited. This also means overlay extensions are not supported, so your options are the <a href="https://developer.mozilla.org/en-US/Add-ons/SDK/Tutorials/Mobile_development" target="_blank">SDK</a> and bootstrapped extensions.</p> <p>Since mobile applications have very limited screen real estate, browsers need to maximize the content area, meaning extensions can&#8217;t add buttons or toolbars to the browser. However, the Firefox for Android team recently developed the <a href="https://developer.mozilla.org/en-US/Add-ons/Firefox_for_Android/Firefox_Hub_Walkthrough" target="_blank">Firefox Hub API</a> that lets extensions add their content to the home page. This is a smart place to put your extension UI if you need it.</p> <p>You can learn more about Firefox for Android extensions and get help here:</p> <ul> <li><a href="https://developer.mozilla.org/en-US/Add-ons/Firefox_for_Android" target="_blank"><strong>Documentation on MDN.</strong></a></li> <li><a href="https://forums.mozilla.org/addons/viewforum.php?f=7" target="_blank">Add-ons forum.</a></li> <li>#extdev and #mobile channels on <a href="https://wiki.mozilla.org/IRC" target="_blank">Mozilla IRC</a>.</li> </ul> <h3>Overlay extensions</h3> <p><a href="https://developer.mozilla.org/en-US/Add-ons/Overlay_Extensions" target="_blank">Overlay extensions</a> are the old way of making add-ons. It might still make sense for you to use this approach if you need to create a very complex overlay or have other specific needs. However, having to restart Firefox to install this kind of extension is annoying because it breaks the user&#8217;s workflow, so you should definitely consider using the SDK or bootstrapped extensions first.</p> <p>You can get help about extension development in general here:</p> <ul> <li><strong><a href="https://developer.mozilla.org/en-US/Add-ons" target="_blank">Documentation on MDN.</a></strong></li> <li><a href="https://blog.mozilla.org/addons/" target="_blank">Articles in this blog.</a></li> <li><a href="https://forums.mozilla.org/addons/viewforum.php?f=7" target="_blank">Add-ons forum.</a></li> <li>#extdev channel on <a href="https://wiki.mozilla.org/IRC" target="_blank">Mozilla IRC</a>.</li> </ul>       </div><!-- .entry-content -->    <footer
