I have decided that it is better not to start the project with building  a shell for the cockpit , but instead begin with  more essential part -  the panel.  <br>This full-scale Baron-58 panel is fully controlled by single Arduino board through built in X-Plane UDP. All buttons, switches, encoders is workable excluding a couple of them nonfunctional in the X-Plane.</p>  <h2>Interface - Arduino</h2> <p>For the I/O interface  I wanted  to use well-known open source hardware  -<a href="http://arduino.cc"> Arduino</a> microcontroller platform that looked like a good choice for making the interface. User-friendly software development environment (IDE), a lot of material on the Internet for developers. The library includes a variety of modules to control servos, stepper motors, analog sensors, gyroscope sensors etc., that  make it easy to organize  any I/O system. <a href="intface.html">&nbsp;>>&nbsp;Interface - Arduino&nbsp;>>>&nbsp;</a> <p>  <h2>System configuration:</h2> <p><img alt="" src="pics/sysdiag.jpg" width="950" height="600"><br><span class="comment">Block diagram of the simulator:</span></p> <p>Main computer - Phenom II X4 965, 8GB memory, video-card GigaByte GeForce GTX 570 1280MB, OS - Linux Mint 10-13, 64-bit (<a href="http://linuxmint.com/">linuxmint.com/</a>) <br> (Why Linux? It is a separate issue, but, in short, I have long gone from Windows, and X-Plane fortunately is multi-platform, and, in my experience, in Linux it runs faster (in frames/sec).  </p> <p>This computer  is  "master", with X-Plane installed and used to provide  the external view.</p> <p>Second computer - it is motherboard with videocard and small HDD, mounted inside  of the panel:</p> <p>- AMD Sempron 150, 2900MHz/45Watt, 2GB memory, Video - GeForce GTS250/512M, OS - Linux <a href="http://lubuntu.net/">Lubuntu 64-bit LXDE (lxde.org/)</a></p> <p>Both the computers and Arduino are connected to the network via  router. </p>  <h2>Panel</h2> <p>Second (panel) computer has two LCD panels. Everything is set up so that you only need to press the power button - at startup the whole instrument panel is expanding to fit the work area of both LCD screens</p> <p>For instruments now we are  using our standalone program (still in development), that is receiving data via network and the whole system can be consist of several instrument modules, using cheap old computers or laptops.(no need of the another copy of X-Plane and powerful PC)</p>  <p>But initially (as a temporary solution), for  the instruments  visualization another copy of X-Plane was used, when main computer sends data  via UDP for the second ("Menu: Net Connections-->External Visual" to display  instruments panel)</p> <p> Some data the main PC sends to Arduino (12 DATA groups). Arduino is using them to control LED indicators and servo gauges, and  sends the control parameters to main computer, altogether about 150 of them.</p>  <div class="line"></div> <p style="width:85%; margin-left:5%">When I was planning this project, I downloaded a lot of photos of the Baron cockpit, and then I calculated all sizes  in the graphic editor, drawing upon all the known dimensions, such as cutouts for 3 1/8" instruments and sizes of the GPS, ADF etc.<br> So I got all the dimensions, which, I think, very close to the original (+/- 1 cm at full width).</p> <p style="text-align:center"><img id="plan" src="tech/plan.gif" onClick="Show('plan', '', 'tech/plan.gif')" > <br><span class="photo"> <img src="tech/plan1t.gif" onClick="Show('plan', '', 'tech/plan1.gif')" > <img src="tech/plan2t.gif" onClick="Show('plan', '', 'tech/plan2.gif')" > <img src="tech/plan3t.gif" onClick="Show('plan', '', 'tech/plan3.gif')" > <img src="tech/plan4t.gif" onClick="Show('plan', '', 'tech/plan4.gif')" > <img src="tech/plan5t.gif" onClick="Show('plan', '', 'tech/plan5.gif')" > <img src="tech/plan6t.gif" onClick="Show('plan', '', 'tech/plan6.gif')" > <img src="tech/plan7t.gif" onClick="Show('plan', '', 'tech/plan7.gif')" > <img src="tech/plan8t.gif" onClick="Show('plan', '', 'tech/plan8.gif')" > </span> </p> <p> Total time spent on handwork only, from scratch to the "test flight" stage, was roughly 100 man-hours during one year period of building.</p> <p>&nbsp;</p> <div class="line"></div>  <p style="width:85%;
