<TD> <H5>Errors like this may appear without a valid SSL certificate when visiting sites on HTTPS.</H5> </TD> </TR> </TBODY> </TABLE> <P>&nbsp;</P> <H1 class="firstHeading gumax-firstHeading">Possible causes / recommended actions&nbsp;</H1> <DIV><HR /></DIV> <UL> <LI><EM>Missing a valid SSL certificate.<BR /><BR /></EM>UniFi relies on HTTPS for extra security. This means that the browser will check for valid certificates when making a secure connection to the web server. Although the alert message may prove annoying, there's no risk to the connecting user. To avoid this error:<BR /><BR /><OL> <LI>Buy a signed SSL certificate from any web hosting provider.</LI> <LI>Then make the following changes to the controller:<BR /><BR /> <PRE>sudo su - # cd &lt;unifi_base&gt;  # on Windows, "%USERPROFILE%/Ubiquiti Unifi" cd /usr/lib/unifi   # create new certificate (with csr) java -jar lib/ace.jar new_cert &lt;hostname&gt; &lt;company&gt; &lt;city&gt; &lt;state&gt; &lt;country&gt;  # your CSR can be found at /var/lib/unifi # - unifi_certificate.csr.der # - unifi_certificate.csr.pem  # have this CSR signed by a CA, you'll get a few certificates back... # copy the signed certificate(s) to &lt;unifi_base&gt;  # import the signed certificate and other intermediate certificates java -jar lib/ace.jar import_cert &lt;signed_cert&gt; [&lt;other_intermediate_root_certs&gt;...]</PRE> <P>&nbsp;</P> </LI> </OL></LI> </UL> <P>&nbsp;</P> <P>&nbsp;</P> <P>&nbsp;</P> </DIV></div>                         </div>
