<p>&nbsp;</p> <h4>General Information</h4> <p>&nbsp;</p> <ul> <li> <p>The AccuWeather API provides subscribers access to location based weather data via a simple <a title="Representational State Transfer" target="_self" href="glossary#rest">RESTful</a> web interface</p> </li> <li> <p>Access to the AccuWeather API requires an API key.&nbsp; Contact <strong>sales@accuweather.com</strong> to receive an API key.</p> </li> <li> <p>A partner code will be provided to you by your AccuWeather representative. &nbsp;Please append the assigned partner code to the links provided in the output. &nbsp;<a href="partner-code" title="Append partner code to links provided in output">Click here for an example</a>. &nbsp;</p> </li> <li> <p>Data is available in more than 40<a target="_self" title="List of language and dialect codes" href="languages"> languages and dialects</a></p> </li> <li> <p>Use hostname: <strong>apidev.accuweather.com&nbsp;</strong>for <strong>development</strong> purposes only.</p> </li> <li> <p>Use hostname: <strong>api.accuweather.com</strong> for <strong>production</strong> purposes only</p> </li> <li> <p>Data responses are returned in <a target="_self" title="JavaScript Object Notation" href="glossary">JSON</a></p> </li> <li> <p>SSL (Secure Sockets Layer) data encryption is also available for secure communication with your set up. This is done with the use of "https://" scheme in your URLs.</p> </li> <li> <p>JSONP is available. Please use "parse_response" as the callback parameter</p> </li> <li> <p>If using our<strong> Legacy API</strong> to obtain weather data, please <a title="Documentation about the Legacy API and how to combine it with features from the new API" href="legacyApi">click here for documentation&nbsp;</a></p> </li> </ul> <p>&nbsp;</p> <h4>Status Codes</h4> <p>&nbsp;</p> <ul> <li> <p>Requests will return an appropriate status code.&nbsp;</p> </li> <li> <p>It is possible to suppress the error codes by passing<strong> allowError=false</strong> in the query.&nbsp; In that case, the status will always be 200, barring a 500 error from the server itself.&nbsp;</p> </li> <li> <p>An error response will describe the specifics of any errors.</p> </li> </ul> <table border="0" cellpadding="4" cellspacing="0" width="900"> <tbody> <tr bgcolor="#4682b4"> <td><strong>Error Code</strong></td> <td><strong>Error Name</strong></td> <td><strong>Description</strong></td> </tr> <tr bgcolor="#eeeeee"> <td>200</td> <td>OK</td> <td>Request was fulfilled</td> </tr> <tr bgcolor="#ffffff"> <td>400</td> <td>Bad Request</td> <td>Request had bad syntax or the parameters supplied were invalid</td> </tr> <tr bgcolor="#eeeeee"> <td>403</td> <td>Forbidden</td> <td>Valid API Key was not supplied in the query</td> </tr> <tr bgcolor="#ffffff"> <td>404</td> <td>Not Found</td> <td>Server has not found a route matching the given URI</td> </tr> <tr bgcolor="#eeeeee"> <td>500</td> <td>Internal Error</td> <td>Server encountered an unexpected condition which prevented it from fulfilling the request</td> </tr> </tbody> </table> <p>&nbsp;</p> <p>&nbsp;</p> <h4>Resources</h4> <p>&nbsp;</p> <h5>Locations</h5> <ul> <li>Locations may be cities, points of interest, postal codes, countries, administrative areas (i.e. states and provinces) or regions.&nbsp;</li> </ul> <h5>Current Conditions</h5> <ul> <li>Current weather observations for a particular location. &nbsp;Most recent and historical current condition observations are available for given locations.&nbsp;</li> </ul> <h5>Alerts</h5> <ul> <li>Severe weather alerts for given locations. &nbsp;</li> </ul> <h5>Daily Indices</h5> <ul> <li>Daily indices for a particular location. &nbsp;Daily indices can be retrieved in 1, 5, 10, or 15 day intervals. &nbsp;&nbsp;</li> </ul> <h5>Weather Alarms</h5> <ul> <li>Weather alarms for a particular location. &nbsp;Weather alarms can be retrieved in 1, 5, 10, 15 or 25 day intervals. &nbsp;&nbsp;</li> </ul> <h5>Climo</h5> <ul> <li>Climatology data for a particular location. &nbsp; Climatology records, normals and actuals can be retrieved for a specific date or specific year/month for a given location. &nbsp;&nbsp;</li> </ul> <h5>Daily Forecasts</h5> <ul> <li>Daily forecasts for a particular location. &nbsp; Daily forecasts can be retrieved in 1, 5, 10, 15 or 25 day intervals.&nbsp;</li> </ul> <h5>Hourly Forecasts</h5> <ul> <li>Hourly forecasts for a particular location. &nbsp;Hourly forecasts can be retrieved in 1, 12, 24, 72, 120, or 240 hour intervals.&nbsp;</li> </ul> <h5>Imagery</h5> <ul> <li>Imagery for a particular location. &nbsp;Radar and satellite images of varying resolutions (480x480, 317x237, 640x480, 640x640, 800x800, 1024x1024) are available for most locations.&nbsp;</li> </ul> <h5>Translations</h5> <ul> <li>Translations for a particular group. &nbsp;Groups return a full set of translations for a specific language.</li> </ul> <h5>LocalWeather</h5> <ul> <li>Weather data for a particular location. Includes current conditions, 12 hour forecasts, 10 day forecasts and radar and satellite images.</li> </ul> <p></p> </article></div>             </div><!--/.span -->     </div><!--/.row -->       <hr />      <footer id="footer">             <div id="footer-sig">                 <div class="zone zone-footer"><div class="credits">         <a href="http://accuweather.com"><img src="Media\Default\accuweather_logotype_color_72.jpg" width=150 height=200  /></a>     </div> </div>             </div>     </footer> </div><!--/.container --> </div> <script src="/developers/Modules/Orchard.jQuery/scripts/jquery-1.7.1.min.js" type="text/javascript"></script> <script src="/developers/Themes/Bootstrap/scripts/bootstrap.min.js" type="text/javascript"></script>  </body> </html>
