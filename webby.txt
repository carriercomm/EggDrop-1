<small><i>(September 2010)</i></small></span></td> </tr> </table> <p><b><a href="/wiki/AmigaOS" title="AmigaOS">AmigaOS</a></b> is the proprietary native <a href="/wiki/Operating_system" title="Operating system">operating system</a> of the <a href="/wiki/Amiga" title="Amiga">Amiga</a> personal computer. Since its introduction with the launch of the <a href="/wiki/Amiga_1000" title="Amiga 1000">Amiga 1000</a> in 1985 there have been four major versions and several minor revisions of the operating system.</p> <p>Initially the Amiga operating system had no strong name and branding, as it was simply considered an integral part of the Amiga system as a whole. Early names used for the Amiga operating system included "CAOS" and "AmigaDOS".<sup id="cite_ref-In_the_Beginning_Was_CAOS_1-0" class="reference"><a href="#cite_note-In_the_Beginning_Was_CAOS-1"><span>[</span>1<span>]</span></a></sup> Another non-official name was "Workbench", from the name of the Amiga <a href="/wiki/Desktop_environment" title="Desktop environment">Desktop environment</a>, which was included on a floppy disk named "Amiga Workbench".<sup id="cite_ref-Workbench_Disk_1.0_2-0" class="reference"><a href="#cite_note-Workbench_Disk_1.0-2"><span>[</span>2<span>]</span></a></sup></p> <p>Version 3.1 of the Amiga operating system was the first version to be officially referred to as "Amiga OS" (with a space between "Amiga" and "OS")<sup id="cite_ref-Name_of_the_Amiga_Operating_System_3-0" class="reference"><a href="#cite_note-Name_of_the_Amiga_Operating_System-3"><span>[</span>3<span>]</span></a></sup><sup id="cite_ref-Amiga_OS_3.1_4-0" class="reference"><a href="#cite_note-Amiga_OS_3.1-4"><span>[</span>4<span>]</span></a></sup> by Commodore, possibly inspired by Apple renaming the Mac operating system from "System" to "<a href="/wiki/Mac_OS" title="Mac OS">Mac OS</a>".</p> <p>Version 4.0 of the Amiga operating system was the first version to be branded as a less generic "AmigaOS" (without the space).<sup id="cite_ref-Name_of_the_Amiga_Operating_System_3-1" class="reference"><a href="#cite_note-Name_of_the_Amiga_Operating_System-3"><span>[</span>3<span>]</span></a></sup></p> <p>What many consider the first versions of AmigaOS (Workbench 1.0 up to 3.0) are here indicated with the Workbench name of their original disks.</p> <p></p> <div id="toc" class="toc"> <div id="toctitle"> <h2>Contents</h2> class="tocnumber">8</span> <span class="toctext">References</span></a></li> </ul> </div> <p></p> <h2><span class="mw-headline" id="Kickstart.2FWorkbench_1.0.2C_1.1.2C_1.2.2C_1.3">Kickstart/Workbench 1.0, 1.1, 1.2, 1.3</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS_versions&amp;action=edit&amp;section=1" title="Edit section: Kickstart/Workbench 1.0, 1.1, 1.2, 1.3">edit</a><span class="mw-editsection-bracket">]</span></span></h2> <div class="thumb tright"> <div class="thumbinner" style="width:202px;"><a href="/wiki/File:Amiga_Workbench_1_0.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/b/b3/Amiga_Workbench_1_0.png/200px-Amiga_Workbench_1_0.png" width="200" height="125" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/en/thumb/b/b3/Amiga_Workbench_1_0.png/300px-Amiga_Workbench_1_0.png 1.5x, //upload.wikimedia.org/wikipedia/en/b/b3/Amiga_Workbench_1_0.png 2x" data-file-width="400" data-file-height="250" /></a> <div class="thumbcaption"> <div class="magnify"><a href="/wiki/File:Amiga_Workbench_1_0.png" class="internal" title="Enlarge"></a></div> Workbench 1.0</div> </div> </div> <p>Workbench 1.0 was released for the first time in October 1985. <sup id="cite_ref-history_5-0" class="reference"><a href="#cite_note-history-5"><span>[</span>5<span>]</span></a></sup> The 1.<i>x</i> series of <i>Workbench</i> defaults to a distinctive blue and orange color scheme, designed to give high contrast on even the worst of television screens (the colors can be changed by the user). Version 1.1 consists mostly of bug fixes and, like version 1.0, was distributed only for the <a href="/wiki/Amiga_1000" title="Amiga 1000">Amiga 1000</a>. The entire Workbench operating system consisted of three floppy disks: <i>Kickstart</i>, <i>Workbench</i> and <i>ABasic</i> by <a href="/wiki/MetaComCo" title="MetaComCo">MetaComCo</a>.</p> <p>The Amiga 1000 needed a <i>Kickstart</i> disk to be inserted into floppy drive to boot up. An image of a simple illustration of a hand on a white screen, holding a blue Kickstart floppy, invited the user to perform this operation. After the kickstart was loaded into a special section of memory called the writable control store (WCS), the image of the hand appeared again, this time inviting the user to insert the Workbench disk.</p> <p><i>Workbench</i> version 1.2 was the first to support Kickstart stored in a <a href="/wiki/Read-only_memory" title="Read-only memory">ROM</a>. A <i>Kickstart</i> disk was still necessary for Amiga 1000 models; it was no longer necessary for Amiga 500 or 2000, but the users of these systems had to change the ROMs (which were socketed) to change the <i>Kickstart</i> version.</p> <p><i>Workbench</i> now spanned two floppy disks, and supported installing and booting from hard drive (assuming the Amiga was equipped with one), the name of the main disk was still named "Workbench" (which is also the user interface portion of the operating system). The second disk was the <i>Extras</i> disk. The system now shipped with <i><a href="/wiki/AmigaBasic" title="AmigaBasic" class="mw-redirect">AmigaBasic</a></i> by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>, the only software Microsoft ever wrote for the Amiga.</p> <p><i>Kickstart</i> version 1.2 corrected various flaws and added <a href="/wiki/AutoConfig" title="AutoConfig" class="mw-redirect">AutoConfig</a> support. AutoConfig is a protocol similar to and is the predecessor of <a href="/wiki/Plug_and_Play" title="Plug and Play" class="mw-redirect">Plug and Play</a>, in that it can configure expansion boards without user intervention.</p> <p><i>Kickstart</i> version 1.3 improved little on its predecessor, the most notable change being auto booting from hard drives. <i>Workbench 1.3</i>, on the other hand, users can find several significant improvements to Workbench, including <a href="/wiki/Amiga_Fast_File_System" title="Amiga Fast File System">FFS</a> a faster file system for hard disks storage which resolved the problem of <a href="/wiki/Amiga_Old_File_System" title="Amiga Old File System">old Amiga filesystem</a> which wasted too much hard disk space due to the fact it could store only 488 bytes in any block of 512 bytes keeping 24 bytes for checksums. Many improvements were made to the CLI (command line interface) of Amiga which was now a complete text based Shell, named <i>AmigaShell</i>, and various additional tools and programs.</p> <h2><span class="mw-headline" id="Kickstart.2FWorkbench_1.4">Kickstart/Workbench 1.4</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS_versions&amp;action=edit&amp;section=2" title="Edit section: Kickstart/Workbench 1.4">edit</a><span class="mw-editsection-bracket">]</span></span></h2> <p>Kickstart/Workbench 1.4 was a <a href="/wiki/Software_testing#Beta_testing" title="Software testing">beta</a> version of the upcoming 2.0 update and never released, but the Kickstart part was shipped in very small quantities with early <a href="/wiki/Amiga_3000" title="Amiga 3000">Amiga 3000</a> computers, where it is often referred to as the "<a href="/w/index.php?title=Superkickstart&amp;action=edit&amp;redlink=1" class="new" title="Superkickstart (page does not exist)">Superkickstart</a> ROM". In these machines it is only used to bootstrap the machine and load the Kickstart that will be used to actually boot the system. The appearance of a very early first release of 1.4 was similar to 1.3, but with colors slightly changed. A second version was similar to that of 2.0 and higher, with just minor differences. It is, however, possible to dump out of the OS selection screen by clicking where one would expect to see a close gadget. This will cause the machine to boot Kickstart 1.4 using either the wb_2.x: partition, or from a floppy.</p> <h2><span class="mw-headline" id="Workbench_2.0.2C_2.04.2C_2.05.2C_2.1">Workbench 2.0, 2.04, 2.05, 2.1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS_versions&amp;action=edit&amp;section=3" title="Edit section: Workbench 2.0, 2.04, 2.05, 2.1">edit</a><span class="mw-editsection-bracket">]</span></span></h2> <div class="thumb tright"> <div class="thumbinner" style="width:202px;"><a href="/wiki/File:Wb_20.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/0/0e/Wb_20.png/200px-Wb_20.png" width="200" height="138" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/en/thumb/0/0e/Wb_20.png/300px-Wb_20.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/0/0e/Wb_20.png/400px-Wb_20.png 2x" data-file-width="640" data-file-height="442" /></a> <div class="thumbcaption"> <div class="magnify"><a href="/wiki/File:Wb_20.png" class="internal" title="Enlarge"></a></div> Workbench 2.0</div> </div> </div> <p>Workbench 2.0 was released in 1990<sup id="cite_ref-history_5-1" class="reference"><a href="#cite_note-history-5"><span>[</span>5<span>]</span></a></sup> and introduced a lot of improvements and major advances to the GUI of the overall Amiga operating system. The harsh blue and orange colour scheme was replaced with a much easier on the eye grey and light blue with 3D aspect in the border of the windows. The Workbench was no longer tied to the 640×256 (PAL) or 640×200 (NTSC) display modes, and much of the system was improved with an eye to making future expansion easier. For the first time, a standardised "look and feel" was added. This was done by creating the <i>Amiga Style Guide</i>, and including libraries and software which assisted developers in making conformant software. Technologies included the GUI element creation library <i>gadtools</i>, the software installation scripting language <i>Installer</i>, and the <i>AmigaGuide</i> hypertext help system.</p> <p>Workbench 2.04 introduced ARexx, a system-wide scripting language. Programmers could add so-called "ARexx ports" to their programs, which allowed them to be controlled from ARexx scripts. Using ARexx, you could make two completely different programs from different vendors work together seamlessly. For example, you could batch-convert a directory of files to thumbnail images with an ARexx-capable image-manipulation program, create and index HTML table of the thumbnails linking to the original images, and display it in a web browser, all from one script. ARexx became very popular, and was widely adopted by programmers.</p> <p>The AmigaDOS, previously written in <a href="/wiki/BCPL" title="BCPL">BCPL</a> and very difficult to develop for beyond basic file manipulation, was mostly rewritten in <a href="/wiki/C_programming_language" title="C programming language" class="mw-redirect">C</a>.</p> <p>Unfortunately, some badly written software – especially games – failed to run with 2.x, and so a lot of people were upset with this update. Most often, the failure occurred because programmers had used directly manipulated private structures maintained by the operating system, rather than using official function calls. Many users circumvented the problem by installing so-called "kickstart switchers", a small circuit board which held both a Kickstart 1.3 and 2.0 chip, with which they could swap between Kickstart versions at the flick of a switch.</p> <p>2.x shipped with the A500+ (2.04), A600 (2.05), A3000 and A3000T. Workbench 2.1 was the last in this series, and only released as a software update. It included useful features such as <a href="/wiki/CrossDOS" title="CrossDOS">CrossDOS</a>, to support working with floppy disks formatted for <a href="/wiki/IBM_PC_compatible" title="IBM PC compatible">PCs</a>. Since 2.1 was a software-only release, there was no Kickstart 2.1 ROM.</p> <p>2.x also introduced PCMCIA card support, for the slot on the A600.</p> <p>Workbench 2.1 introduced also a standard hypertext markup language for easily building guides for the user or help files, or manuals. It was called <a href="/wiki/AmigaGuide" title="AmigaGuide">AmigaGuide</a>. Release 2.1 was also the first Workbench release to feature a system-standard localization system, allowing the user to make an ordered list of <i>preferred languages</i>; when a locale-aware application runs, it asks the operating system to find the <i>catalog</i> (a file containing translations of the application's strings) best matching the user's preferences.</p> <h2><span class="mw-headline" id="AmigaOS_3.0.2C_3.1">AmigaOS 3.0, 3.1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS_versions&amp;action=edit&amp;section=4" title="Edit section: AmigaOS 3.0, 3.1">edit</a><span class="mw-editsection-bracket">]</span></span></h2> <div class="thumb tright"> <div class="thumbinner" style="width:202px;"><a href="/wiki/File:Amiga_Workbench_3_screenshot.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/7/79/Amiga_Workbench_3_screenshot.png/200px-Amiga_Workbench_3_screenshot.png" width="200" height="160" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/en/thumb/7/79/Amiga_Workbench_3_screenshot.png/300px-Amiga_Workbench_3_screenshot.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/7/79/Amiga_Workbench_3_screenshot.png/400px-Amiga_Workbench_3_screenshot.png 2x" data-file-width="557" data-file-height="445" /></a> <div class="thumbcaption"> <div class="magnify"><a href="/wiki/File:Amiga_Workbench_3_screenshot.png" class="internal" title="Enlarge"></a></div> Amiga Workbench 3</div> </div> </div> <p>Workbench 3.0 was released in 1992 and version 3.1 between 1993 (for the [Amiga CD32|CD32]) and 1994 (for other Amiga models). Version 3.1 is the last AmigaOS released by Commodore. <sup id="cite_ref-history_5-2" class="reference"><a href="#cite_note-history-5"><span>[</span>5<span>]</span></a></sup></p> <p>The 3.x series added support for new Amiga models. Other new features included:</p> <ul> <li>A universal data system, known as DataTypes, that allowed programs to load pictures, sound, text and other content in formats they didn't understand directly, through the use of standard plugs (<i>see</i> <a href="/wiki/Object-oriented_operating_system" title="Object-oriented operating system">object-oriented operating system</a>)</li> <li>Better color remapping for high-color display modes and support for new <a href="/wiki/Amiga_Advanced_Graphics_Architecture" title="Amiga Advanced Graphics Architecture">AGA</a> chipset</li> <li>Improved visual appearance for Workbench desktop</li> <li>CD-ROM support as required for <a href="/wiki/Amiga_CD32" title="Amiga CD32">Amiga CD32</a> (3.1)</li> </ul> <p>3.x shipped with the <a href="/wiki/Amiga_CD32" title="Amiga CD32">CD32</a>, <a href="/wiki/Amiga_1200" title="Amiga 1200">A1200</a>, <a href="/wiki/Amiga_4000" title="Amiga 4000">A4000</a> and <a href="/wiki/Amiga_4000T" title="Amiga 4000T">A4000T</a>.</p> <h2><span class="mw-headline" id="AmigaOS_3.5.2C_3.9">AmigaOS 3.5, 3.9</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS_versions&amp;action=edit&amp;section=5" title="Edit section: AmigaOS 3.5, 3.9">edit</a><span class="mw-editsection-bracket">]</span></span></h2> <div class="thumb the Amiga operating system by its new owners. Whereas all previous OS releases ran on <a href="/wiki/Motorola_68000" title="Motorola 68000">Motorola 68000</a> new release 3.5 onwards required a <a href="/wiki/Motorola_68020" title="Motorola 68020">68020</a> or better, CD-ROM and at least 4 MB RAM. The Kickstart 3.1 was also required as the operating system didn't include new ROM.</p> <p>Updates included:</p> <ul> <li>Supplied with <a href="/wiki/Internet_protocol_suite" title="Internet protocol Workbench 4.0<sup class="reference" id="ref_Fair_Use"><a href="#endnote_Fair_Use">[5]</a></sup></div> </div> </div> <p>A new version of AmigaOS was released on December 24, 2006 after five years of development by <a href="/wiki/Hyperion_Entertainment" title="Hyperion Entertainment">Hyperion Entertainment</a> (<a href="/wiki/Belgium" title="Belgium">Belgium</a>) under license from <a href="/wiki/Amiga,_Inc" title="Amiga, Inc" class="mw-redirect">Amiga, Inc</a>. for <a href="/wiki/AmigaOne" title="AmigaOne">AmigaOne</a> registered users.</p> <p>During the five years of development, users of <a href="/wiki/AmigaOne" title="AmigaOne">AmigaOne</a> machines could download from Hyperion repository Pre-Release Versions of AmigaOS 4.0 as long as these were made available. As witnessed by many users into Amiga discussion forum sites, these versions were stable and reliable, despite the fact that they are technically labeled as "pre-releases".</p> <p>Last stable version of AmigaOS 4.0 for AmigaOne computers is the "July 2007 Update", released for download 18 July 2007 to the registered users of AmigaOne machines.<sup class="reference" id="ref_AmigaOS_4.0_first_update"><a href="#endnote_AmigaOS_4.0_first_update">[1]</a></sup></p> <p>AmigaOS 4 Classic was released commercially for older Amiga computers with CyberstormPPC and BlizzardPPC accelerator cards in November 2007. It had previously been available only to developers and beta-testers.</p> <h3><span class="mw-headline" id="Version_4.0">Version 4.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS_versions&amp;action=edit&amp;section=7" title="Edit section: Version 4.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3> <p>The new version is <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a>-native, finally abandoning the <a href="/wiki/Motorola" title="Motorola">Motorola</a> <a href="/wiki/68k" title="68k" class="mw-redirect">68k</a> <a href="/wiki/Central_processing_unit" title="Central processing unit">processor</a>. AmigaOS 4.0 will run on some <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a> hardware, which currently only includes <a href="/wiki/Amiga_1200" title="Amiga 1200">A1200</a>, <a href="/wiki/Amiga_3000" title="Amiga 3000">A3000</a> and <a href="/wiki/Amiga_4000" title="Amiga 4000">A4000</a> with PowerPC accelerator boards and <a href="/wiki/AmigaOne" title="AmigaOne">AmigaOne</a> motherboards. Amiga, Inc.'s distribution policies for AmigaOS 4.0 and any later versions require that for third-party hardware the OS must be bundled with it, with the sole exception of Amigas with Phase 5 PowerPC accelerator boards, for which the OS will be sold separately.</p> <p>AmigaOS 4.0 Final introduced a new memory system based on the <a href="/wiki/Slab_allocator" title="Slab allocator" class="mw-redirect">slab allocator</a>.</p> <p>Features, among others:</p> <ul> <li>Fully skinnable GUI</li> <li>Virtualized memory</li> <li>Integrated viewer for PDF and other document formats</li> <li>Support for PowerPC (native) and 68k (interpreted/JIT) applications</li> <li>New drivers for various hardware</li> <li>New memory allocation system</li> <li>Support for file sizes larger than 2 GB</li> <li>Integrated <a href="/wiki/ReTargetable_Graphics#Picasso_96" title="ReTargetable Graphics">Picasso 96</a> 2D Graphics API</li> <li>Integrated <a href="/wiki/Warp3D" title="Warp3D">Warp3D</a> 3D Graphics API</li> </ul> <p><br /></p> <h3><span class="mw-headline" id="Version_4.1">Version 4.1</span><span class="mw-editsection"><span
